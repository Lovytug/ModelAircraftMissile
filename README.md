# AircraftMissile
Прототип программной архитектуры на C++17 для моделирования полёта управляемой ракеты в составе зенитного комплекса.

Проект был разработан в рамках производственной практики как концептуальная реализация программной части для будущего полунатурного стенда.

Основной задачей было спроектировать архитектуру системы, которая позволит в дальнейшем добавлять новые компоненты моделирования (физику, алгоритмы управления, модели среды и т.д.).

_Важно: данный репозиторий представляет собой архитектурный прототип, а не готовую систему моделирования._


## Цель проекта
Продемонстрировать возможный подход к построению _расширяемой системы моделирования на языке c++17_

В рамках проекта решались задачи:
  - разработка модульной архитектуры
  - разделение компанентов системы на: физику, алгоритмы, среда, модель ракеты и ее компаненты
  - создание базовых абстракций для последующего расширения системы
  - использование чистого c++17 без внешних фреймворков
Выполнение данных задач способствует для дальнейшего добавления новых компанент и моделей


## Статус проекта

Проект находится в состоянии _архитектурного прототипа_.

Код не дорабатывался после завершения производственной практики и представлен в том виде, в котором он использовался для демонстрации архитектурного подхода.

Основная ценность проекта — демонстрация архитектуры и принципов организации модульной системы на C++17.


## Архитекутра проекта
Проект разделен на несколько основных модулей
```
algorithm/        алгоритмы управления и наведения
environment/      модель окружающей среды
physics/          физические расчёты и модели
rocket/           модель ракеты
math_lib/         математические утилиты
base_component/   базовые абстаркции для компонентов системы
type_data/        структуры данных
python_explore/   исследовательские ноутбуки и предэксперименты
```


## Основные идеи проектирования
### Модульность
Каждая часть системы вынесена в отдельный модуль

### Расширяемость
Архитектура предлагает возможность добавления новых компанентов:
  - алгоритмов управления
  - моделей среды
  - физических моделей
  - элементов системы и под системы
Также архитектура позоляет засчет абстракций легко подменять одни алгоритмы/физические движки на другие, не ломая логику

### Особеннось
Проект реализован преймущественно на c++17


## Ограничения проекта
Данный репозиторий не является полноценной системой моделирования
Текущая реализация представляет собой частичный образ архитектуры, в которой содержатся около реальные, с точки зрения математики поведения, модели, но с некоторыми упрощениями. 

Также проект не имеет грфической визуализации, подразумивая, что все расчеты и значения будут записываться в файл, для последующего анализа в других фреймах/языках/иное.


## Используемый стек
- **с++17**
- **стандартная библиотека STL**
- **Python+Jupyter для проведения предэкспериментов поведения физики**
